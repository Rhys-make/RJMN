# 基于预训练语言模型的简历岗位匹配度评估与面试辅助系统课程设计报告

## 1. 绪论

在线招聘场景中，简历与岗位 JD 的匹配评估是一类典型的自然语言处理任务。传统关键词匹配和 TF-IDF 方法主要依赖词面重叠，难以处理“关键词不同但语义相近”的情况。预训练语言模型能够学习文本的上下文语义表示，更适合用于岗位要求和简历片段之间的语义匹配。

本项目不是从零训练大模型，而是采用预训练语言模型进行语义表示和生成式分析。这样能够在有限课程周期内利用已有模型能力完成高质量 NLP 应用系统。

## 2. 需求分析

系统需要支持简历和 JD 的文本输入、文件上传、示例数据加载、语义匹配、技能实体抽取、多维评分、生成式分析、TF-IDF baseline 对比以及 JSON/Markdown 报告导出。系统还必须在模型加载失败时保持稳定运行，并明确显示模板降级模式。

## 3. 系统总体架构

系统采用 Streamlit 前端和 Python 核心模块实现。整体流程如下：

1. 用户输入简历和 JD。
2. `file_reader.py` 读取 txt、pdf、docx 文件。
3. `preprocess.py` 清洗文本，并切分简历 section 和 JD requirement。
4. `skill_extractor.py` 提取技能实体。
5. `embedding_model.py` 加载 BGE 预训练语义模型并生成向量。
6. `evidence_matcher.py` 计算 JD requirement 与简历 evidence 的相似度。
7. `scoring.py` 计算四维综合匹配分。
8. `llm_generator.py` 使用 Qwen2.5 生成分析，失败则模板降级。
9. `baseline_tfidf.py` 计算 TF-IDF baseline。
10. `app.py` 展示结果并支持报告导出。

## 4. 关键技术

### 4.1 预训练语言模型

项目使用 Hugging Face 上的预训练语言模型，不进行大模型训练或微调。语义匹配模型为 `BAAI/bge-small-zh-v1.5`，生成式模型为 `Qwen/Qwen2.5-1.5B-Instruct`。

### 4.2 BGE 语义向量模型

BGE 模型用于将 JD requirement 和简历 evidence 编码为语义向量。语义相近的文本在向量空间中距离更近，因此系统可以识别“表达不同但含义接近”的匹配关系。

### 4.3 Qwen2.5 生成式模型

Qwen2.5 负责根据简历、JD、匹配分数、命中技能、缺失技能和 evidence 生成候选人优势、短板、简历优化建议、面试问题和回答思路。Prompt 明确要求模型只根据给定信息分析，不允许编造简历中没有的信息。

### 4.4 余弦相似度

系统使用余弦相似度衡量文本向量之间的接近程度：

```text
cosine(A, B) = (A · B) / (||A|| × ||B||)
```

### 4.5 TF-IDF baseline

TF-IDF baseline 用于传统词面匹配对比。它不作为主方法，只用于实验区展示传统方法与 BGE 语义模型的差异。

### 4.6 多维度评分模型

综合匹配度公式为：

```text
综合匹配度 =
技能实体匹配分 × 30%
+ 语义证据匹配分 × 40%
+ 项目经历匹配分 × 20%
+ 软技能匹配分 × 10%
```

匹配等级：

- 85-100：高度匹配
- 70-84：较高匹配
- 50-69：一般匹配
- 0-49：匹配度较低

## 5. 系统实现

系统实现分为文本读取、预处理、技能抽取、BGE 语义匹配、证据检索、四维评分、Qwen 生成式分析和前端展示。模型加载失败时，`embedding_model.py` 和 `llm_generator.py` 都会返回清晰错误信息，不会导致 Streamlit 页面崩溃。

## 6. 实验设计

实验数据位于 `data/eval_cases.json`，包括高度匹配、较高匹配、一般匹配、低匹配、语义相近但关键词不同、关键词相同但语义弱匹配 6 类样例。

## 7. 实验结果与分析

系统展示 TF-IDF baseline 分数和 BGE 语义模型分数。对于同义表达和语义改写样例，BGE 模型更容易找到合理简历证据；对于只有关键词堆砌但缺乏项目证据的样例，逐项 evidence 匹配可以降低误判风险。

## 8. 消融实验

### 8.1 TF-IDF baseline

仅使用 TF-IDF 时，系统依赖词面重叠，优点是速度快，缺点是语义理解能力弱。

### 8.2 BGE 预训练语义匹配

加入 BGE 后，系统能够对 JD requirement 和简历 evidence 进行语义级匹配，提升对同义表达的识别能力。

### 8.3 BGE + Qwen2.5 生成式分析

加入 Qwen2.5 后，系统可以将结构化匹配结果转换为自然语言建议和面试问题。如果 Qwen 不可用，则使用模板降级，保证系统稳定运行。

## 9. 系统不足

1. 简历和 JD 的结构化切分仍基于规则。
2. BGE 和 Qwen 首次运行需要下载模型。
3. CPU 环境下 Qwen 生成速度可能较慢。
4. 技能词典覆盖范围有限，需要继续扩展。

## 10. 部署与交接说明

为了便于课程小组协作，项目增加了 `docs/AI_HANDOFF.md` 作为交接文档，并提供 `scripts/check_models.py` 用于检查本地 BGE 与 Qwen 模型是否可用。

项目支持优先从本地目录加载模型：

```text
models/bge-small-zh-v1.5/
models/Qwen2.5-1.5B-Instruct/
```

其中 BGE 用于主语义匹配链路，Qwen 用于生成式分析。若 Qwen 权重缺失，系统会显示 `template_fallback`，但这只是生成模块降级，不影响 BGE 语义证据匹配和四维评分主流程。

交接时需要重点检查：

1. `scripts/check_models.py` 是否显示 BGE 可用。
2. Qwen 目录是否包含 `model.safetensors`。
3. `streamlit run app.py` 是否能打开页面。
4. 示例数据是否能完整输出四维评分、证据匹配、关键词分析和报告导出。

## 11. 总结与展望

本项目实现了一个以 BGE 语义匹配为主方法、Qwen2.5 生成式分析为辅助的简历岗位匹配系统。系统兼顾可解释性、稳定性和课程展示效果，后续可以扩展批量简历排序、岗位领域词典和更强的 reranker 模型。
